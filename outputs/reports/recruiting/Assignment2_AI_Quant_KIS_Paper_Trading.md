# 과제2. AI 활용 검증 자료

## KOSPI200 AI 퀀트 전략 및 KIS API 기반 시스템 트레이딩 dry-run

제가 직접 만든 산출물은 KOSPI200 종목을 대상으로 한 AI 퀀트 투자전략과 KIS API 연동 시스템 트레이딩 실행 자료입니다. 결과물은 `outputs/paper_trading/latest_target_portfolio.csv`, `outputs/paper_trading/latest_order_plan.csv`, `outputs/reports/KIS_Paper_Trading_Run_Report.md`로 저장했습니다. KIS 상시모의투자 서버에 연결해 현재가와 모의계좌 잔고를 조회했고, 1,000만원 모의계좌 기준 목표 포트폴리오와 주문전표를 생성했습니다.

## 무엇을 만들었는가

전략은 KOSPI200 유니버스에서 매월 말 상위 30개 종목을 선별하는 구조입니다. 데이터는 KIS API 기반 2007~2026년 일별 가격 데이터와 TS2000 재무데이터를 결합했습니다. 재무데이터에서는 Value, Quality, Growth 팩터를 만들고, 가격데이터에서는 Momentum과 Low Volatility 팩터를 만들었습니다. 여기에 Ridge 회귀 기반 ML 모델로 다음 1개월 초과수익률을 예측해 위성 신호로 결합했습니다.

최종 전략은 시장 국면에 따라 주식 비중을 조절합니다. 2026년 6월 30일 신호 기준 시장 국면은 neutral로 분류했고, 목표 주식 비중은 80%, 현금 비중은 20%로 설정했습니다. KIS 상시모의투자 잔고 조회 결과 현금 1,000만원 계좌로 확인되었고, 계좌 규모상 고가 종목은 목표수량이 0으로 제외되었습니다. 최종 주문전표는 총 21건, 약 493.7만원 규모의 매수 주문으로 생성되었습니다.

상위 목표 포트폴리오에는 SK하이닉스, LG이노텍, 삼성전기, 대우건설, 미스토홀딩스, 삼성전자, SK스퀘어, BNK금융지주, DB하이텍, SK 등이 포함되었습니다. 실제 모의주문 전송도 시도했으나, 실행 시점이 장 종료 이후라 KIS 서버가 "모의투자 장종료" 응답을 반환했습니다. 따라서 이번 산출물은 전략 신호 생성, 계좌 조회, 주문수량 산출, 주문 API 호출 및 오류 응답 처리까지 검증한 시스템 트레이딩 실행 자료입니다.

## 어떤 도구를 사용했는가

사용 도구는 Python, pandas, KIS Open API, TS2000 재무데이터, 그리고 AI 코딩 보조 도구입니다. Python으로 데이터 로딩, 팩터 계산, 백테스트, ML 예측, 주문수량 계산을 구현했습니다. KIS API는 기존 가격 수집뿐 아니라 현재가 조회와 계좌/주문 API 구조까지 확장했습니다. 안전을 위해 주문 전송은 기본값에서 비활성화했고, `--submit-paper-orders` 옵션을 명시적으로 켠 경우에만 모의투자 주문을 전송할 수 있도록 했습니다. 또한 실계좌 설정에서는 주문 전송이 막히도록 `KIS_IS_PAPER=true` 조건을 추가했습니다.

제가 AI에 사용한 대표 프롬프트는 다음과 같습니다.

`KOSPI200 멀티팩터 전략을 KIS API 기반 모의투자 시스템으로 확장하려고 한다. 기존 factor score 파일에서 최신 신호일 상위 30종목을 뽑고, KIS 현재가를 조회한 뒤, 1억원 기준 목표비중과 주문수량을 계산하는 dry-run 모듈을 설계해줘. 실제 주문은 기본적으로 나가지 않아야 하며, 주문 전표와 실행 리포트를 CSV/Markdown으로 저장해야 한다.`

이 프롬프트를 바탕으로 AI가 제안한 구조를 그대로 사용한 것이 아니라, 기존 프로젝트의 파일 구조와 데이터 스키마에 맞게 직접 수정했습니다. 예를 들어 전략 점수는 `institutional_core_satellite_scores.csv`를 사용했고, 가격 fallback은 `prices_2007_2026.csv`를 사용했습니다. KIS 현재가 조회가 실패할 경우 로컬 가격으로 대체하고, KIS 잔고가 없거나 사용하지 않을 경우 모의 현금을 넣어 주문전표를 생성하도록 했습니다.

## 어떻게 풀었는가

첫째, 투자 아이디어를 수치화했습니다. Value, Quality, Growth, Momentum, Low Volatility를 각각 표준화하고, 시장 국면별로 Balanced, Value-Momentum, Defensive, ML 신호의 비중을 다르게 적용했습니다. 둘째, 검증 구간을 Train 2007~2016, Validation 2017~2021, Test 2022~2026으로 나누어 데이터 누수를 줄였습니다. 셋째, 백테스트에서는 월간 리밸런싱, 거래비용 10bp, Top 30 동일가중을 적용했습니다. 기존 검증 결과에서 ML 전략은 Test 구간 CAGR 32.31%, Sharpe 1.14, IR 1.01을 기록했고, 통합 멀티팩터 전략도 Test 구간 CAGR 27.21%, Sharpe 1.09, IR 0.73을 기록했습니다.

마지막으로 이 전략을 실제 운용 프로세스에 가깝게 바꾸었습니다. 신호 생성 후 KIS 현재가를 조회하고, 목표 비중을 원화 주문금액으로 변환하고, 현재 보유수량과 목표수량의 차이를 주문수량으로 계산했습니다. 먼저 dry-run으로 주문전표를 검증한 뒤 `--submit-paper-orders` 옵션으로 KIS 상시모의투자 주문 API 호출까지 진행했습니다. 실행 시점이 장 종료 이후라 체결은 발생하지 않았지만, API 주문 요청과 오류 응답 처리까지 확인했습니다. 이 방식은 운용역이 투자 아이디어를 제시했을 때, 주니어가 Python으로 빠르게 데이터 검증, 포트폴리오 산출, 주문 전표화까지 연결하는 업무 흐름과 유사합니다.

이번 산출물의 핵심은 AI를 "투자 판단 대체 도구"가 아니라 "리서치와 자동화 생산성을 높이는 도구"로 사용했다는 점입니다. 최종 판단은 데이터, 백테스트, 리스크 통제, 주문 전표 검증을 통해 사람이 확인하도록 설계했습니다.
